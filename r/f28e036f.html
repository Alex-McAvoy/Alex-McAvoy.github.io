VSCode 配置 R 语言环境

发表于 2021-01-10 分类于 R语言 阅读次数：
本文字数： 2.2k 阅读时长 ≈ 2 分钟

【概述】

对于 R 语言的用户，大多数都选择 RStudio，但在习惯使用 VSCode 后，使用 RStudio 不仅不习惯，而且其可定制性很差

VSCode 是由微软开发的一款通用 IDE，除了更高的颜值之外，还支持几乎所有主流语言，其具有代码高亮、代码补全、自定义主题和快捷键、Git 支持等专业特性

通过大量的扩展插件，VSCode 可以完成很多不同的任务，其通用性可以保证在一个编辑器之内完成各种不同语言的项目，提高工作效率

R 语言作为一门小众语言，其插件并不是官方开发，而是由第三方开发者所贡献，这导致配置一个尚可的 R 环境需要从四处安装不同的东西，才能开始工作

需要的插件有：

  • R support for Visual Studio Code
  • R LSP Client

此外，还需要 radian 库来作为替代终端

【R for VSCode】

R for VSCode 是 R 语言的核心插件，用于运行 R 文件或所选行，具有语法高亮，基础代码片段和代码执行等功能

安装完成后，便可以通过快捷键（默认为 Ctrl + Enter）将编辑器中的所选行发送至 R Console 运行，但有很多第三方包命令无法补全

【R LSP Client】

R LSP Client 提供了 R 语言的代码补全、查看函数定义以及参数预览等功能，可以获得更好的自动补全、查看帮助文档等特性，以满足 R 语言中第三方包代码的自动补全

但安装了这个插件，会发现自动补全功能依然没有出现，实际上，插件只是在编辑器上提供了实现 LSP 的条件，而在 R 语言一侧还需要另外 languageserver 包来完成搭接

【radian 库】

使用 R 语言时，大多数时间是将编辑器中的代码发送到 Console 中运行，有时也要直接在 Console 中运行一些代码以进行快速交互

RStudio 为 Console 提供了语法高亮、自动补全等功能的加持，使得在 Console 中也可以有比较方便的输入输出体验

但在 R for VSCode 插件默认在 Terminal 中调用 R 进程，会返回朴素的 R Console 界面，代码高亮、自动补全不在考虑范围之内

radian 弥补了这个缺点，为 Terminal 增加了代码高亮、自动补全、多行输入等特殊功能，radian 是一个 python 项目，需要安装 python 环境后再安装这个 module

这是因为 R for VSCode 在开启一个 R 进程时，默认带入了三个选项 --no-save、--no-restore、--no-site-file，而 radian 将 --no-save 和 --no-restore 作为默认选项，因此无法再传入

因此，在设置中关闭 --no-save 和 no-restore 两个选项即可，找到 Rterm: Option，将这两项删掉即可
