Linux——目录的相关操作

发表于 2018-06-14 | 分类于 Linux
字数统计: 833 字 | 阅读时长 ≈ 3 分钟

【目录的基本操作】

cd 命令

cd 命令是 Change Directory 的缩写，用于切换工作目录，其语法为：cd [相对路径或绝对路径]

在切换目录时，有以下比较特殊的目录：

  • .：本级目录
  • ..：上一级目录
  • ./：等价于 .
  • ../：等价于 ..
  • -：前一个工作目录
  • ~：当前用户身份所在的主文件夹
  • ~account：account 用户的主文件夹（account 为用户名）

需要说明的是，当仅给出 cd 未给出目录时，代表着返回主文件夹。

pwd 命令

pwd 命令是 Print Working Directory 的缩写，即显示当前所在的工作目录的绝对路径，其语法为：pwd [-P]

-P 选项会显示出当前的路径，而非使用连接(link)的路径。

mkdir 命令

mkdir 命令是 Make Directory 的缩写，用于创建新目录，其语法为：mkdir [-mp] 目录

当需要创建多层目录时，需要加上 -p 选项，来递归的进行创建。

当要创建具有某些权限的目录时，需要加上 -m 选项，并且利用数字来说明权限，如果没有加上 -m 选项，系统会使用默认权限。

rmdir 命令

rmdir 命令是 Remove Directory 的缩写，用于删除目录，要求被删除的目录不能存在其他目录与文件，其语法为：mkdir [-p] 目录

当要删除多层目录时，需要加上 -p 选项，来递归的进行删除，要求被删除的目录中不能存在文件。

【环境变量】

概述

查看文件属性的 ls 命令的绝对路径为：/bin/ls，在任何地方执行这个命令，就一定可以显示某些信息而不会说找不到 /bin/ls 命令，这是因为环境变量 PATH 的帮助。

在执行一个命令的时候，系统会依照 PATH 的设置去每个 PATH 定义的目录下查询文件名为 ls 的可执行文件，如果在 PATH 定义的目录中含有多个文件名为 ls 的可执行文件，那么先被查询到的同名命令先被执行。

同理，对于其他命令来说也是如此。

变量内容

PATH 这个变量对于不同的用户内容不同，其内容是由一堆目录组成的，每个目录中间由冒号 : 隔开，且每个目录间是有顺序之分的。

对 PATH 变量的操作

临时修改

要想添加某个路径到环境变量中，使用 export 命令进行赋值即可，其语法为：export PATH=$PATH:新路径

想要删除环境变量中的某个路径时，将其他的路径复制后利用 export 命令重新赋值给 PATH 即可。

需要说明的是，使用上述两种方法进行添加和删除时，在关闭终端后会失效。

永久修改

PATH 变量是在 .bashrc 这个配置文件中指定的，如果想要永久的加入或删除某个路径，只能通过修改这个配置文件来完成。

其设置步骤如下：

  1. 通过 touch ~/.bashrc 创建 .bashrc
  2. 通过 start ~/.bashrc 进入 .bashrc
  3. 在 .bashrc 中写入以下内容：export PATH=路径
  4. 通过 source ~/.bashrc 执行 .bashrc 文件
