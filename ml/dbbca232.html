【欠拟合与过拟合】

概念

泛化（Generalize），是指一个假设模型应用到新样本的能力。

偏差（Bias），又称为表观误差，是指个别测定值与测定的平均值之差，在机器学习中，偏差描述的是根据样本拟合出的模型输出结果与真实结果的差距。

方差（Variance），在概率论中是衡量随机变量或一组数据时离散程度的度量，在机器学习中，方差描述的是样本上训练出的模型在测试集上的表现。

拟合（Fitting），是指模型能否很好地描述样本，具有比较好的泛化能力。

欠拟合（Under Fitting），是指样本不够或模型具有高偏差，不够精确，测试样本的特性没有学习到，不具备泛化性，拿到测试集后无法准确地去预测。

过拟合（Over Fitting），是指模型具有高方差，太过贴近于训练数据的特征，此时代价函数可能十分接近于 0，模型过于庞大或变量过多，使得模型过度依赖于训练数据，在训练集上近乎完美地预测了所有的数据，但无法处理未知问题，在测试集上表现平平。

解决方法

过拟合问题与欠拟合问题是所有机器学习算法都要考虑的问题。

对于欠拟合问题来说，其比较容易克服，常见解决方法有：

  • 增加新的特征值，加入特征组合、高次特征等，以增大假设空间
  • 添加多项式特征，例如：线性回归模型的多项方程
  • 使用非线性模型，例如：决策树、SVM 等模型
  • 减少正则化参数，正则化的目的是用来防止过拟合的，但当模型出现欠拟合时，需要减少正则化参数

而对于过拟合问题来说，常见的解决方法有：

  • 适当的减少特征值，人工检查特征变量来决定特征变量的去留
  • 使用模型选择算法，自动选择特征变量的去留
  • 加入正则化参数，保留所有的特征变量，但减少量级或减少 $\theta$ 参数的大小

【正则化】

引入

正则化是为了解决过拟合问题的一种方法。

举例来说，我们有一组数据，我们利用高阶多项式进行拟合，假设函数为 $h_{\theta}(x) = \theta_0 + \theta_1 x + \theta_2 x^2 + \theta_3 x^3 + \theta_4 x^4$ 时，会得到如下一个十分曲折的图像：

很明显，出现了过拟合的情况，虽然我们可以人为的去除一些特征项，但若想保留所有特征的高阶项，可以按照如下方法来进行优化。

由于我们的目标是最小化代价函数 $J(\theta)=\frac{1}{2n}\sum_{i=1}^n(h_{\theta}(x^{(i)})-y^{(i)})^2$，而要使得这个函数尽可能小，就要使得 $\theta_3,\theta_4$ 参数尽可能的小，因此我们可以在其前加一个惩罚项，即加一个较大的数，使得参数值尽可能的小，例如：$J(\theta)=\frac{1}{2n}\sum_{i=1}^n(h_{\theta}(x^{(i)})-y^{(i)})^2+1000\theta_3^2 + 1000\theta_4^2$

这样一来，由于 $\theta_3,\theta_4$ 参数前有了一个较大的系数，使得函数整体比较大，如果我们要使代价函数整体尽可能的小，那么可以预见到的是 $\theta_3,\theta_4$ 参数会很小，接近于零，如同直接去掉了 $\theta_3x^3,\theta_4x^4$ 这两项。

那么，假设函数相当于一个二次函数： $h_{\theta}(x) = \theta_0 + \theta_1 x + \theta_2 x^2$，最后拟合出来的数据实际上与一个二次函数相差无几，只是加上了两个很小的、接近于 0 的项 $\theta_3x^3,\theta_4x^4$

上述的这个过程，就是正则化的思想，即：为所有的 $\theta$ 参数加上一个惩罚项，来尽量去简化假设模型。

代价函数

当特征变量 $x$ 与 $\theta$ 参数多达 $m+1$ 个时，我们并不知道该选出哪些参数来加入惩罚项从而缩小他们的值，因此在正则化过程中，我们要做的，就是对代价函数进行修改，令其缩小除 $\theta_0$ 外的 $m$ 个 $\theta$ 参数。

可以看出，与线性回归模型原来的代价函数相比，多加了一个额外的正则化项 $\lambda \sum_{j=1}^m \theta_j^2$ 来缩小每个参数的值。

正则化参数

$\lambda$ 称为正则化参数，其用于控制两个不同目标间的平衡关系，一方面我们要更好地去拟合训练集，另一方面要保持参数尽可能的小，即控制 $\sum_{i=1}^n (h_{\theta}(x^{(i)})-y^{(i)})^2$ 与 $\sum_{j=1}^m \theta_j^2$ 的取舍。

如果 $\lambda$ 设置的过小，那么 $\theta$ 参数的惩罚程度为起到应有的效果，仍会出现过拟合问题。

如果 $\lambda$ 设置的过大，那么 $\theta$ 参数的惩罚程度太大，所有的 $\theta$ 参数几乎全部等于 0，这就使得把假设函数的除去 $\theta_0$ 的项全都忽略掉了，相当于直接用 $h_{\theta}(x)=\theta_0$ 来去拟合数据，这就出现了欠拟合问题。

因此，为了让正则化起到应用的效果，我们应该去选择一个合适的正则化参数 $\lambda$
