Ⅲ.单链表

i）基本概念

1.定义：线性表的链式存储方式，通过一组任意存储单元存储线性表中的数据元素

2.特点：

1）要耗费一定空间存放指针：每个结点除存放数据元素（数据域）外，还要存储指向下一结点的指针（指针域）

2）容量改变方便：由于不要求大片连续空间，可以利用指针来扩充空间

3）非随机存取：由于指针的特性，查找某个元素时，要从表头开始遍历

3.头指针：用于标识单链表，始终指向单链表的第一个结点

4.头结点

1）方式：在单链表第一个结点前附加一头结点，数据域一般用于存储表长等信息，指针域指向单链表第一个结点

2）引入目的：编程方便

①头结点指针域指向单链表第一个结点，使得第一个结点的操作与其他结点操作一致，无须特殊处理

②无论单链表是否为空，头指针都指向头结点的指针，空表与非空表的操作统一，无须特殊处理

3）与头指针关系：带头结点时，头指针指向头结点（第 0 个结点），头结点指向链表第 1 个结点；不带头结点时，头指针直接指向单链表的第 1 个结点

ii）结点类型

说明：

  • struct LNode 别名：`LNode``
  • `struct LNode * 等价于：Lnode *，强调是一个结点
  • struct LNode * 别名：LinkList，强调是一个单链表

iii）初始化（无参构造函数）

1.带头结点：分配一个头结点，令头指针指向头结点

2.不带头结点：令头指针直接指向空表

iv）基本操作

1.判空：判断 L 是否为空

1）带头结点：判断 L 头指针指向的头结点是否指向 NULL

2）不带头结点：判断 L 头指针是否指向 NULL

2.求表长：获取 L 长度

1）带头结点：从头结点开始，判断下一个结点是否为 NULL

2）不带头结点：从头指针开始，判断当前结点是否为 NULL

3.输出：从头遍历 L，依次输出各结点数据域

1）带头结点：从头结点指向的第 1 个结点开始

2）不带头结点：从头指针指向的第1个结点开始

v）查找

1.按值查找：获取 L 中结点数据域与 elem 值相同的结点位置，从头到尾遍历即可

1）带头结点：从头结点指向的第1个结点开始

2）不带头结点：从头指针指向的第1个结点开始

2.按位查找：获取 L 中第 i 个位置的结点

1）带头结点：越界判断与扫描器从 0 开始

2）不带头结点：越界判断与扫描器从 1 开始

vi）插入

1.后插：在给定 p 结点后插入元素 elem，用于按位序插入，不可单独使用

2.前插：在 p 结点前插入元素 elme，用于按位序插入，不可单独使用**

由于指针机制是后向机制，若采用循环遍历的方式找到前驱结点进行前插，时间复杂度为 $O(n)$，而采用以下交换数据的方式，时间复杂度仅为 $O(1)$

3.按位序插入：在 L 中的第 i 个位置插入元素 elem，可以找到第 i-1 个结点后插，也可以找到第 i 个结点前插

1）带头结点：找到第 i-1 个结点 p 后，申请新结点 s，改变链接关系

2）不带头结点：插入第1个位置需要改变头指针 L，插入其他位置时与带头结点的插入相同

vii）删除

1.删除指定结点：删除 L 中指定的结点 p，并返回删除结点的值 elem

1）步骤

①若删除的是非最后结点，只需将要删除的结点 p 之后的结点 s 的数据存入 p，然后再断开 p 与 s 的链接，令 p 指向 s 之后的结点，再释放空间即可，时间复杂度为 $O(1)$

②若删除的是最后结点，由于其没有后继结点，无法采用上述方法，只能从头到尾开始遍历，在单链表中匹配与 p 地址相同的结点，然后将该结点前的结点指向 NULL，再释放空间，时间复杂度为 $O(n)$

2）应用：该方法结合按值查找、按位查找方法，寻找到结点后进行删除

viii）单链表的建立（有参构造函数）

1.尾插法：利用尾指针 tail，每次将新申请的结点插到终端结点的后面，始终让新结点在最后的位置

1）带头结点：建立头结点后，利用尾指针，不断申请新结点，填充数据域后建立后继关系即可

2）不带头结点：由于初始头指针指向为空，因此要先建立一个初始结点，将头指针指向该结点，再对剩下的 n-1 个元素建立结点，填充数据域建立后继关系

2.头插法：每次将新申请的结点插到头结点的后面，即始终让新结点在第一的位置，常应用于链表的逆置

1）带头结点：每次都是对头结点进行后插操作

2）不带头结点：实质是建立后继关系后，通过不断交换头指针与新结点的地址来实现头插
