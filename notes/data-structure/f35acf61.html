顺序表

发表于 2019-03-11 | 分类于 学习笔记 ， 数据结构
字数统计: 1,939 字 | 阅读时长 ≈ 9 分钟

【】

顺序表具有以下特点：

  • 无需为表示结点间的逻辑关系而增加额外的存储空间
  • 可随机读取表中的任一元素
  • 插入或删除运算需要移动大量元素
  • 表长变化较大时，难以确定合适的存储规模

i）基本概念

1.定义：逻辑上相邻元素，物理存储单元位置也相邻，元素间关系由存储单元邻接关系体现

2.特点

1）支持随机访问：通过 data[i-1] 即可访问第 i 个元素，在 $O(1)$ 的时间复杂度内即可完成

2）存储密度高：只存储数据元素

3）扩展容量不方便：静态分配方式无法扩展，动态分配方式扩展要消耗大量的时间

4）插入、删除数据元素不方便

ii）分配方式与初始化

1.静态分配：表长声明后无法更改

2.动态分配：表长声明后可更改

说明：

①在动态分配时，malloc 与 new 用于分配空间，free 与 delete 用于回收空间

②malloc 与 free 搭配使用， new 与 delete 搭配使用

③malloc/free 是 C/C++ 标准库函数，new/delete 是 C++ 操作符

④malloc 要手动计算类型大小，返回类型为 void* （需要强制转换成所需类型）；new 自动计算类型大小，返回对应类型指针

⑤从系统层面来说，真正开出空间的是 malloc，new/delete 的实现依赖于内存管理接口

⑥扩充线性表长度还可利用 realloc() 函数

⑦为加深存储空间的开辟回收的理解，本文所有代码均用 malloc/free 实现

⑧由于 template 与 typedef 不能共用，本文所有元素类型设置为 int

iv）查找

1.按值查找：获取 L 中与 elem 值相同的元素位置，返回 0 代表查找失败

1）实现

2）时间复杂度分析

①最好时间复杂度：查找元素在表头 i=1，需比较 1 次，$O(1)$

②最坏时间复杂度：查找元素在表尾 i=n ，需比较 n 次，$O(n)$

③平均时间复杂度：查找元素在第 i 个位置的概率为 $p_i=\frac{1}{n}$，在各位置上需要执行后移语句 $i$ 次，平均移动次数为 $\sum_{i=1}^{n+1}p_i*i=\frac{n+1}{2}$ 次，$O(n)$

2.按位查找：获取 L 中第 pos 个位置的元素值，返回 0 代表查询失败

1）实现

2）时间复杂度分析：支持随机访问，$O(1)$

v）插入与删除

1.插入：在顺序表 L 上第 i 个位置插入新元素 elem

说明：关于 pos 位置与数组位置，数组索引从 0 开始，线性表索引从 1 开始，因此 pos-1 为对应数组位置

2）时间复杂度分析

①最好时间复杂度：在表尾 i=n+1 插入，后移语句不执行，$O(1)$

②最坏时间复杂度：在表头 i=1 插入，后移语句执行 $n$ 次，$O(n)$

③平均时间复杂度：在第 i 个位置插入概率为 $p_i=\frac{1}{n+1}$，在各位置上需要执行后移语句 $n-i+1$ 次，平均移动次数为 $\sum_{i=1}^{n+1}p_i*(n-i+1)=\frac{n}{2}$ 次，$O(n)$

②最坏时间复杂度：在表头 i=1 删除，后移语句执行 $n$ 次，$O(n)$

③平均时间复杂度：在第 i 个位置删除概率为 $p_i=\frac{1}{n}$，在各位置上需要执行后移语句 $n-i$ 次，平均移动次数为 $\sum_{i=1}^{n+1}p_i*(n-i)=\frac{n-1}{2}$ 次，$O(n)$
