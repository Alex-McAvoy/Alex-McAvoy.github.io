操作系统——操作系统的结构

发表于 2019-09-13 | 分类于 学习笔记 ， 学习笔记——操作系统

【传统的 OS 结构】

1.无结构 OS

早期的 OS，主要是为了功能的实现与提高效率，此时的 OS 是为数众多的一组过程的集合。

每个过程可以任意地相互调用其他过程，没有一个清晰的结构，内部的结构复杂又混乱，而且程序错误很多，调试、维护困难。

2.模块化结构 OS

在 20 世纪 60 年代，出现了一种结构化程序设计技术，基于分解与模块化原则来控制大型软件的复杂度。

OS 不再是由众多的过程直接构成，而是按其功能精心地划分为若干个具有一定独立性的模块，每个模块具有某方面的功能，各模块间通过接口实现交互，然后再进一步将各模块分为若干具有一定功能的子模块。

利用模块化来设计 OS，不仅提高了设计的正确性、可理解性和可维护性，还增强了 OS 的可适应性，并加快了开发过程，此外，如果要增减、修补某些功能，可选用相应功能模块。

3.分层次结构 OS

分层次结构 OS 是在模块化 OS 的基础上引入了有序分层法，即在目标系统 $A_0$ 和裸机 $A_n$ 之间铺设若干中间层 $A_{n-1},A_{n-2},…,A_2,A_1$，使得 $A_n$ 通过 $A_{n-1},A_{n-2},…,A_2,A_1$ 层，最终能在 $A_0$ 上运行。

在 OS 中，常采用自底向上法来铺设这些中间层：每一步设计都建立在可靠的基础上，即每一层仅能使用其底层所提供的功能和服务。

分层次结构 OS 能够保证系统正确性，且容易扩充和维护，但由于分层单向依赖，执行一个功能时，总要自下而上地穿越多层，增加系统通信开销，降低了效率。

【微内核 OS 结构】

微内核 OS 结构是在 20 世纪 80 年代后期发展起来的，由于其能有效地支持多处理机运行，故非常适用于分布式系统环境。

微内核 OS 结构以 OS 为核心，以 C/S 为基础，采用了面向对象程序设计方法，微内核是一个可在其上构建通用 OS 的基础内核，该内核程序运行在核心态，开机常驻内存。

微内核并非一个完整的 OS，其与硬件紧密相关，只是实现了较基本功能，且应用了机制与策略分离原理，其中，机制是实现某一功能的具体执行机构，位于基层；策略是指借助某些参数和算法实现功能优化或达到不同功能目标，位于系统高层。

微内核具有灵活、可选配功能服务器模块的优点，且内核是一个稳定的核心，只有它能直接访问硬件，可以保证服务器在用户态出错时绝不会导致系统瘫痪，此外，其具有可移植性强的特点。
